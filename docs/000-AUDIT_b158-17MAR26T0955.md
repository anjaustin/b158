## Overview

This repository contains Microsoft's BitNet.cpp framework (1-bit/1.58-bit LLM inference) along with experimental research on ordered dithering for LLM inference and a reflective architecture called "Halo."

---

## Novel

### 1. Ordered Dithering for LLMs
**Location**: `BitNet-archives/BitNet/eval/paper/ORDERED_DITHERING_RESEARCH.md`

Applying Bayer matrix dithering (from digital halftoning) to logit space before token selection. Counterintuitively **reduces entropy** rather than increasing it—creating deterministic preferences that focus ambiguous decisions.

### 2. Phase Transition Discovery
**Location**: `BitNet-archives/BitNet/eval/paper/HALO_ARCHITECTURE.md:18-40`

Dithering intensity maps to discrete "cognitive basins" with sharp cliff edges:
- 0.0-0.074: Academic mode
- 0.096-0.143: Curator (List) mode  
- 0.144-0.161: Guide (Prose) mode
- 0.1618+: Philosopher mode

Not gradual blending—strict phase transitions.

### 3. Halo Architecture
**Location**: `BitNet-archives/BitNet/eval/paper/HALO_ARCHITECTURE.md:44-108`

Multi-pass generation structure:
```
DIVERGE (intensity=0.16) → SYNTHESIZE (0.13) → CONVERGE (0.10)
```
Creates outputs that inherit from both parent modes—termed "cognitive breeding."

### 4. Fundamental Constant Bounds
**Location**: `BitNet-archives/BitNet/eval/paper/ORDERED_DITHERING_RESEARCH.md:91-102`

Intensity range derived from physical/mathematical constants:
- Minimum: α = 1/137 ≈ 0.0073 (fine structure constant)
- Maximum: φ/10 ≈ 0.1618 (golden ratio)

## Useful

### 1. BitNet CPU Inference Framework
**Location**: `BitNet/`

- Speedups: **1.37x-5.07x** on ARM, **2.37x-6.17x** on x86
- Energy reduction: **55.4%-70.0%** (ARM), **71.9%-82.2%** (x86)
- Supports 100B model on single CPU at 5-7 tokens/sec

### 2. Configurable Tiling & Parallelism
**Location**: `BitNet/src/README.md:14-31`

- GEMM block sizes tunable via `include/gemm-config.h`
- Parallelism degree: [2, 4, 8]
- Row block size: [2, 4, 8, 16, 32]
- Column block size: [32, 64, 128, 256, 512, 1024]

### 3. Embedding Quantization
**Location**: `BitNet/src/README.md:113-148`

- Q6_K format reduces memory with <0.04 perplexity increase
- Implemented via `--quant-embd` flag

### 4. GPU Kernel (W2A8)
**Location**: `BitNet/gpu/`

- Custom CUDA GEMV kernel for 2-bit weights × 8-bit activations
- **3x speedup** on NVIDIA A100 vs BF16 baseline
- Weight permutation and dp4a instruction optimizations

### 5. Cross-Architecture Support
- x86 with AVX2
- ARM with NEON
- ARM with DOTPROD extension

### 1. HALO Philosophy
**Location**: `BitNet-archives/BitNet/eval/paper/HALO_ARCHITECTURE.md:286-297`

Poetic framing ("Breathe out (diverge), breathe in (converge)") masks that the core mechanism is simply multi-pass generation with different intensity parameters.

**Location**: `BitNet-archives/BitNet/eval/paper/HALO_ARCHITECTURE.md:138-162`

- [D] Sampling Strategy

- [J] Verification Loop

### 3. Unverified Synthesis Claims
**Location**: `BitNet-archives/BitNet/eval/paper/HALO_ARCHITECTURE.md:107`

Claim that synthesis creates "something neither parent produced alone" lacks rigorous human evaluation benchmarks.

### 4. Pulse Mechanism
**Location**: `BitNet-archives/BitNet/eval/paper/HALO_ARCHITECTURE.md:111-134`

Described as "experimental" with finding that macro-level mode switching requires multi-pass, not intra-pass oscillation—essentially admits limited utility.
